Attachments

Resource for Attachments. Attachments uploaded to the system. Example, pictures are uploaded via this resource and then associated to the article via Article Attachments.

Operations

  • POST rest/v1/attachments/{attachmentId}
  • GET rest/v1/attachments/{attachmentId}
  • GET rest/v1/attachments
  • DELETE rest/v1/attachments/{attachmentId}

Authorization

  • All operations can be performed by clients with a valid Authorization header

Resource

Property Name Format Description
contentType read-only, 150 max length The Content Type of this Attachment, this value extracted when the attachment is uploaded to the system
attachmentId read-only Id associated with this resource
_links read-only

Links for versions of the Attachment (aka essence file).

  • original: the uploaded file in its original form
  • thumbnail: thumbnail generated from the original file
name read-only The name of the Attachment, this value extracted when the attachment is uploaded to the system
