Use Cases

This page describes a typical use case where a User creates a Trade Users submit and match Items and finally generate the TradeResults

Olavo is a User which Authenticates on the system. Note that the returned Authorization header needs to be included in all secured endpoints.

----- Request ----- GET http://localhost:8081/authenticate ----- Response ----- Status: HTTP/1.1 200 Headers: {Authorization: 3764-7777-9277}

Olavo gets his Authentication details. Note that Olavo will need know his User.userId in the future.

----- Request ----- GET http://localhost:8081/rest/v1/authentications/ Headers: {Authorization: 3764-7777-9277}{Content-Type: application/json} ----- Response ----- Status: HTTP/1.1 200 { "userId" : 1, "_links" : [ { "rel" : "self", "href" : "http://localhost:8081/rest/v1/authentications/" } ] }

Olavo creates a Trade, by default he becomes the organizer of the Trade. The Trade.state is SUBMITTING_ITEMS meaning that other Users are allowed to become members and submit Items.

----- Request ----- POST http://localhost:8081/rest/v1/trades/ Headers: {Authorization: 3764-7777-9277}{Content-Type: application/json} { "name" : "Board games in Ottawa" } ----- Response ----- Status: HTTP/1.1 201 { "name" : "Board games in Ottawa", "tradeId" : 1, "state" : "SUBMITTING_ITEMS", "_links" : [ { "rel" : "self", "href" : "http://localhost:8081/rest/v1/trades/1" } ] }

Olavo gets his TradeMemberhip details.

----- Request ----- GET http://localhost:8081/rest/v1/trade-memberships/?tradeId=1&userId=1 Headers: {Authorization: 3764-7777-9277}{Content-Type: application/json} ----- Response ----- Status: HTTP/1.1 200 Headers: {X-Pagination-Total-Count: 1} [ { "tradeMembershipId" : 1, "userId" : 1, "tradeId" : 1, "type" : "OWNER", "_links" : [ { "rel" : "self", "href" : "http://localhost:8081/rest/v1/trade-memberships/1" } ] } ]

Olavo adds two Items to his TradeMemberhip.

----- Request ----- POST http://localhost:8081/rest/v1/trade-memberships/1/items Headers: {Authorization: 3764-7777-9277}{Content-Type: application/json} { "name" : "Pandemic Legacy: Season 1" } ----- Response ----- Status: HTTP/1.1 201 { "name" : "Pandemic Legacy: Season 1", "itemId" : 1, "_links" : [ { "rel" : "self", "href" : "http://localhost:8081/rest/v1/trade-memberships/1/items/1" } ] }
----- Request ----- POST http://localhost:8081/rest/v1/trade-memberships/1/items Headers: {Authorization: 3764-7777-9277}{Content-Type: application/json} { "name" : "Pandemic Legacy: Season 2" } ----- Response ----- Status: HTTP/1.1 201 { "name" : "Pandemic Legacy: Season 2", "itemId" : 2, "_links" : [ { "rel" : "self", "href" : "http://localhost:8081/rest/v1/trade-memberships/1/items/2" } ] }

Maria Authenticates on the system. Note that the returned Authorization header needs to be included in all secured endpoints.

----- Request ----- GET http://localhost:8081/authenticate ----- Response ----- Status: HTTP/1.1 200 Headers: {Authorization: 2501-2908-3849}

Maria gets her Authentication details. Note that Maria will need know her User.userId in the future.

----- Request ----- GET http://localhost:8081/rest/v1/authentications/ Headers: {Authorization: 2501-2908-3849}{Content-Type: application/json} ----- Response ----- Status: HTTP/1.1 200 { "userId" : 2, "_links" : [ { "rel" : "self", "href" : "http://localhost:8081/rest/v1/authentications/" } ] }

Maria subscribes to Olavo's Trade via TradeMemberhip endpoint.

----- Request ----- POST http://localhost:8081/rest/v1/trade-memberships/ Headers: {Authorization: 2501-2908-3849}{Content-Type: application/json} { "tradeId" : 1, "userId" : 2 } ----- Response ----- Status: HTTP/1.1 201 { "tradeMembershipId" : 2, "userId" : 2, "tradeId" : 1, "type" : "MEMBER", "_links" : [ { "rel" : "self", "href" : "http://localhost:8081/rest/v1/trade-memberships/1" } ] }

Maria adds three Items to her TradeMemberhip.

----- Request ----- POST http://localhost:8081/rest/v1/trade-memberships/2/items Headers: {Authorization: 2501-2908-3849}{Content-Type: application/json} { "name" : "Stone Age" } ----- Response ----- Status: HTTP/1.1 201 { "name" : "Stone Age", "itemId" : 3, "_links" : [ { "rel" : "self", "href" : "http://localhost:8081/rest/v1/trade-memberships/2/items/3" } ] }
----- Request ----- POST http://localhost:8081/rest/v1/trade-memberships/2/items Headers: {Authorization: 2501-2908-3849}{Content-Type: application/json} { "name" : "Carcassonne" } ----- Response ----- Status: HTTP/1.1 201 { "name" : "Carcassonne", "itemId" : 4, "_links" : [ { "rel" : "self", "href" : "http://localhost:8081/rest/v1/trade-memberships/2/items/4" } ] }
----- Request ----- POST http://localhost:8081/rest/v1/trade-memberships/2/items Headers: {Authorization: 2501-2908-3849}{Content-Type: application/json} { "name" : "No Thanks!" } ----- Response ----- Status: HTTP/1.1 201 { "name" : "No Thanks!", "itemId" : 5, "_links" : [ { "rel" : "self", "href" : "http://localhost:8081/rest/v1/trade-memberships/2/items/5" } ] }

Olavo changes the Trade.state to MATCHING_ITEMS. Note that only Trade owners can change the Trade.state

----- Request ----- PUT http://localhost:8081/rest/v1/trades/1 Headers: {Authorization: 3764-7777-9277}{Content-Type: application/json} { "name" : "Board games in Ottawa", "state" : "MATCHING_ITEMS" } ----- Response ----- Status: HTTP/1.1 200 { "name" : "Board games in Ottawa", "tradeId" : 1, "state" : "MATCHING_ITEMS", "_links" : [ { "rel" : "self", "href" : "http://localhost:8081/rest/v1/trades/1" } ] }

Olavo matches his Items with Items from other members via the WantItems endpoint.
He offers Pandemic Legacy: Season 1 and wants Maria's Stone Age in exchange.

----- Request ----- POST http://localhost:8081/rest/v1/trade-memberships/1/items/1/want-items Headers: {Authorization: 3764-7777-9277}{Content-Type: application/json} { "itemId" : 3, "priority" : 0 } ----- Response ----- Status: HTTP/1.1 201 { "wantItemId" : 1, "itemId" : 3, "priority" : 0, "_links" : [ { "rel" : "item", "href" : "http://localhost:8081/rest/v1/trade-memberships/1/items/3" }, { "rel" : "self", "href" : "http://localhost:8081/rest/v1/trade-memberships/1/items/1/want-items/3" } ] }

Maria matches her Items with Items from other members via the WantItems endpoint.
She offers Stone Age and wants Olavo's Pandemic Legacy: Season 1 in exchange.
She also offers Stone Age and wants Olavo's Pandemic Legacy: Season 2 in exchange.

----- Request ----- POST http://localhost:8081/rest/v1/trade-memberships/2/items/3/want-items Headers: {Authorization: 2501-2908-3849}{Content-Type: application/json} { "itemId" : 1, "priority" : 0 } ----- Response ----- Status: HTTP/1.1 201 { "wantItemId" : 2, "itemId" : 1, "priority" : 0, "_links" : [ { "rel" : "self", "href" : "http://localhost:8081/rest/v1/trade-memberships/2/items/3/want-items/1" }, { "rel" : "item", "href" : "http://localhost:8081/rest/v1/trade-memberships/2/items/1" } ] }
----- Request ----- POST http://localhost:8081/rest/v1/trade-memberships/2/items/3/want-items Headers: {Authorization: 2501-2908-3849}{Content-Type: application/json} { "itemId" : 2, "priority" : 1 } ----- Response ----- Status: HTTP/1.1 201 { "wantItemId" : 3, "itemId" : 2, "priority" : 1, "_links" : [ { "rel" : "self", "href" : "http://localhost:8081/rest/v1/trade-memberships/2/items/3/want-items/2" }, { "rel" : "item", "href" : "http://localhost:8081/rest/v1/trade-memberships/2/items/2" } ] }

Olavo changes the Trade.state to MATCHING_ITEMS_ENDED.

----- Request ----- PUT http://localhost:8081/rest/v1/trades/1 Headers: {Authorization: 3764-7777-9277}{Content-Type: application/json} { "name" : "Board games in Ottawa", "state" : "MATCHING_ITEMS_ENDED" } ----- Response ----- Status: HTTP/1.1 200 { "name" : "Board games in Ottawa", "tradeId" : 1, "state" : "MATCHING_ITEMS_ENDED", "_links" : [ { "rel" : "self", "href" : "http://localhost:8081/rest/v1/trades/1" } ] }

Olavo gets the TradeResults which says:
Olavo (offering_user_id=1) sends Pandemic Legacy: Season 1 (offering_item_id=1) to Maria (receiving_user_id=2) in exchange for Stone Age (receiving_item_id=3)
Maria (offering_user_id=2) sends Stone Age (offering_item_id=3) to Olavo (receiving_user_id=1) in exchange for Pandemic Legacy: Season 1 (receiving_item_id=1)
Note that any member of a Trade can get TradeResults.

----- Request ----- GET http://localhost:8081/rest/v1/trades/1/results Headers: {Authorization: 3764-7777-9277}{Content-Type: application/json} ----- Response ----- Status: HTTP/1.1 200 offering_user_id,offering_user_name,offering_item_id,offering_item_name,receiving_user_id,receiving_user_name,receiving_item_id,receiving_item_name 1,1508556063105AuthenticationOAuthNewUserMock,1,Pandemic Legacy: Season 1,2,1508556063757AuthenticationOAuthNewUserMock,3,Stone Age 2,1508556063757AuthenticationOAuthNewUserMock,3,Stone Age,1,1508556063105AuthenticationOAuthNewUserMock,1,Pandemic Legacy: Season 1 # ############################# # Summary of Trade [1 : Board games in Ottawa] # Total of items: 5 # Total of traded items: 2 # Total of items not traded: 3 # #############################
