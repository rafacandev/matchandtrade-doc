Authenticate

Endpoint to authenticate on the system. It redirects the client to an oAuth server (e.g.: Google oAuth) and returns with a header called Authorization.

The Authorization header acts as a session token and needs to be included to any secured URL.

Operations

  • GET authenticate
  • GET authenticate/sign-out

Authorization

  • Public

Examples

GET http://localhost:8081/authenticate Status: HTTP/1.1 200
----- Request ----- GET http://localhost:8081/authenticate ----- Response ----- Status: HTTP/1.1 200 Headers: Authorization: 8326-4353-4730 Content-Length: 0 Date: Mon, 06 Nov 2017 00:06:37 GMT
----- Request ----- curl -v GET \ http://localhost:8081/authenticate ----- Response ----- Status: HTTP/1.1 200 Headers: Authorization: 8326-4353-4730 Content-Length: 0 Date: Mon, 06 Nov 2017 00:06:37 GMT
GET http://localhost:8081/authenticate/sign-out Status: HTTP/1.1 205
----- Request ----- GET http://localhost:8081/authenticate/sign-out Headers: Authorization: 8326-4353-4730 ----- Response ----- Status: HTTP/1.1 205 Headers: Set-Cookie: JSESSIONID=B0E752BF0B71BDA788AE4929BE6900F8; Path=/; HttpOnly Date: Mon, 06 Nov 2017 00:06:37 GMT
----- Request ----- curl -v GET \ -H "Authorization: 8326-4353-4730" \ http://localhost:8081/authenticate/sign-out ----- Response ----- Status: HTTP/1.1 205 Headers: Set-Cookie: JSESSIONID=B0E752BF0B71BDA788AE4929BE6900F8; Path=/; HttpOnly Date: Mon, 06 Nov 2017 00:06:37 GMT
