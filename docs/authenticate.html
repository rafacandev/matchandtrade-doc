Authenticate

Endpoint to authenticate on the system. It redirects the client to an oAuth server (e.g.: Google oAuth) and returns with a header called Authorization.

The Authorization header acts as a session token and needs to be included to any secured URL.

Operations

  • GET authenticate
  • GET authenticate/sign-out

Authorization

  • Public

Examples

GET http://localhost:8081/authenticate Status: HTTP/1.1 200
----- Request ----- GET http://localhost:8081/authenticate ----- Response ----- Status: HTTP/1.1 200 Headers: Authorization: 8109-463-5091 Content-Length: 0 Date: Sun, 22 Oct 2017 05:22:50 GMT
......
GET http://localhost:8081/authenticate/sign-out Status: HTTP/1.1 205
----- Request ----- GET http://localhost:8081/authenticate/sign-out Headers: Authorization: 8109-463-5091 ----- Response ----- Status: HTTP/1.1 205 Headers: Set-Cookie: JSESSIONID=50759E6984BBB1753A01AF965108EF58; Path=/; HttpOnly Date: Sun, 22 Oct 2017 05:25:05 GMT
