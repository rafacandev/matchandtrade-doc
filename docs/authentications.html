Authentications

Resource for Authentications. It returns information about the current authenticated session, typically used to get the userId associated with the current session.

Operations

  • GET rest/v1/authentications

Authorization

  • Only clients with a valid Authorization header

Resource

Property Name Format Description
userId read-only The User associated with this authentication, this value is created during the sign-up (first successful Authenticate)

Example

GET http://localhost:8080/matchandtrade-api/v1/authentications/
HTTP/1.1 200 
GET http://localhost:8080/matchandtrade-api/v1/authentications/
-- Headers --
   Authorization: 8330-6411-8579

---- Response ----
HTTP/1.1 200 
-- Headers --
   Content-Type: application/json;charset=UTF-8
   Transfer-Encoding: chunked
   Date: Thu, 14 Feb 2019 02:38:18 GMT
-- Body --
{
  "userId" : 3006
}
curl -v \
-H "Authorization: 8330-6411-8579" \
http://localhost:8080/matchandtrade-api/v1/authentications/

---- Response ----
