Trades

Resource for trades. Trades are the the central components of the system. The user who creates a trade becomes the organizer while other users can subscribe to the trade and become members. Members submit their items through a trade membership. Later the organizer closes the trade and the system generates the results of the trade.

Operations

  • POST rest/v1/trades/
  • PUT rest/v1/trades/{tradeId}
  • GET rest/v1/trades/{tradeId}
  • GET rest/v1/trades?{queryParameters}
  • GET rest/v1/trades/
  • DELETE rest/v1/trades/{tradeId}

Authorization

  • Only authenticated clients.

Resource

Field NameRulesDescription
stateread-write, enumeration (see values below)The state of this trade, states drive how to interact with the trade.
nameread-write, mandatory, unique, 3 min length, 150 max lengthThe name of this trade
tradeIdread-onlyId associated with this resource
State
NameDescription
SUBMITTING_ITEMSDefault state when a trade is created. Indicates that users can join the trade and submit items.
MATCHING_ITEMSUsers can match items in the trade. Other users cannot join the trade nor submit items.
GENERATING_TRADESThe system is generating the trade results.
CLOSEDThe trade is in ready-only mode
Query Parameters
Field NameDescription
nameResources matching Trade.name
_pageSizeSee pagination
_pageNumberSee pagination

Rules

  • When a new trade is created, the user associated to the Authorization header will be the trade organizer.
  • Only the trade organizer can DELETE a trade.
  • Only the trade organizer can PUT a trade.
