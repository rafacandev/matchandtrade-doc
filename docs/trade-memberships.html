TradeMembership

Resource for TradeMemberships. Users who want to subscribe to a Trade need create a TradeMembership.

Operations

  • POST rest/v1/trade-memberships/
  • GET rest/v1/trade-memberships/{tradeMembershipId}
  • GET rest/v1/trade-memberships?{queryParameters}
  • GET rest/v1/trade-memberships/
  • DELETE rest/v1/trade-memberships/{tradeMembershipId}

Authorization

  • All operations can be performed by clients with a valid Authorization header

Resource

Property Name Format Description
userId read-write, mandatory Must belong to an existing User
tradeId read-write, mandatory Must belong to an existing Trade
tradeMembershipId ready-only, auto-generated Id associated with this resource
type read-only, enumeration
  • OWNER the onwer of the tradeId. This resource is created when a Trade is created.
  • MEMBER a member associated to the tradeId. When a new resource is created via POST rest/v1/trade-memberships/ it has TradeMembership.type=MEMBER

Query Parameters

Property Name Description
userId Refers to a User.userId
tradeId Refers to a Trade.tradeId
_pageNumber See pagination
_pageSize See pagination

Rules

  • Cannot delete a TradeMembership.Type=OWNER of an existing Trade
  • Cannot create more than one TradeMembership for the same Trade
  • Users can subscribe only when Trade.State=SUBMITTING_ITEMS
