Users

Resource for Users. Typically, Users are created on the first time they Authenticate in the system.

Operations

  • PUT rest/v1/users/{userId}
  • GET rest/v1/users/{userId}

Authorization

  • All operations can be performed by clients with a valid Authorization header
  • {userId} must be the value associated with the Authentication header. In other words, first the client needs to get the userId from Authentications and pass it in the URL.

Resource

Property Name Format Description
email read-only The email of this User, this value is assigned by the Authentication Authority (e.g.: Google oAuth) during the sign-up process
name read-write, 150 max length The name of this User. This value is firstly assigned by the Authentication Authority (e.g.: Google oAuth)during the sign-up process
userId read-only Id associated with this resource

Query Parameters

Property Name Description
name Resources matching User.name
_pageSize See pagination
_pageNumber See pagination
